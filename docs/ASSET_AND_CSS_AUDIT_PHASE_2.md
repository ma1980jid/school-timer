# فحص ملفات CSS والأصول — المرحلة الثانية

الفرع:

```text
cleanup-production-1.0
```

## الهدف

فحص ملفات التصميم والأصول قبل أي حذف أو دمج، وتحديد الملفات المستخدمة حاليًا والملفات التي يمكن دمجها لاحقًا.

## 1. ملفات CSS مستخدمة ولا تُحذف الآن

### style.css

الحالة: مستخدم وأساسي.

السبب:

- يتم تحميله مباشرة من index.html.
- يحتوي أساس تصميم شاشة العرض.
- يحتوي خلفية شاشة الحاسوب desktop-bg.webp.
- يتحكم في مواضع الشعار والبطاقات والجدول والساعة.

القرار:

```text
لا يُحذف ولا يُنقل.
```

### ticker-fix.css

الحالة: مستخدم ومساند.

السبب:

- يتم تحميله مباشرة من index.html.
- مسؤول عن إصلاح حركة شريط الرسائل.
- يحتوي تنسيقات مهمة للصندوقين النصيين في الهاتف.
- يحتوي تعديلات على current-row في الهاتف.

القرار:

```text
لا يُحذف الآن.
```

ملاحظة مستقبلية:

يمكن دمجه لاحقًا داخل style.css أو داخل ملف viewer.css عند إعادة الهيكلة.

### mobile-current-row-clean.css

الحالة: مستخدم ومساند.

السبب:

- يتم تحميله ديناميكيًا من supabase-config.js في صفحات العرض.
- مسؤول عن تحسين تمييز الحصة الجارية في الهاتف.

القرار:

```text
لا يُحذف الآن.
```

ملاحظة مستقبلية:

يمكن دمجه لاحقًا مع تنسيقات الهاتف في style.css أو viewer-mobile.css.

## 2. ملفات صور أساسية لا تُحذف

### desktop-bg.webp

الحالة: مستخدم.

السبب:

- يتم تحميله كخلفية شاشة الحاسوب.
- موجود ضمن preload في index.html.
- موجود ضمن Service Worker assets.

القرار:

```text
لا يُحذف.
```

### mobile-bg.webp

الحالة: مستخدم.

السبب:

- يتم تحميله كخلفية شاشة الهاتف.
- موجود ضمن preload في index.html.
- موجود ضمن Service Worker assets.

القرار:

```text
لا يُحذف.
```

### icons/school_logo.png

الحالة: مستخدم كأيقونة افتراضية.

السبب:

- مستخدم في روابط favicon و apple-touch-icon في index.html.
- مع ذلك يجب الحذر لأنه كان سببًا في ظهور شعار المدرسة الافتراضية في مدارس أخرى سابقًا.

القرار:

```text
لا يُحذف الآن.
```

ملاحظة مستقبلية:

الأفضل استبداله لاحقًا بأيقونة عامة محايدة لا تخص أي مدرسة.

## 3. ملاحظات تقنية

- ملفات CSS الحالية تعمل بأسلوب patch فوق patch، وهذا طبيعي في مرحلة التطوير.
- في الإنتاج الأفضل إنشاء بنية أوضح:

```text
assets/css/viewer.css
assets/css/viewer-mobile.css
assets/css/dashboard.css
assets/css/system-admin.css
```

- لا يتم دمج ملفات CSS الآن حتى لا يتغير التصميم فجأة.
- أي دمج لتنسيقات الهاتف يجب أن يتم بعد اختبار شاشة الهاتف تحديدًا.

## 4. قرار المرحلة الثانية

لا توجد ملفات CSS أو صور أساسية مرشحة للحذف الآن.

الملفات التي يمكن دمجها لاحقًا فقط:

```text
ticker-fix.css
mobile-current-row-clean.css
```

لكن لا يتم ذلك قبل اختبار بصري كامل.

## 5. الخطوة القادمة

فحص ملفات JavaScript المساندة وتحديد ما يمكن دمجه لاحقًا، مع الإبقاء على الملفات الحالية أثناء مرحلة الاستقرار.
