# توثيق جداول Supabase المستخدمة في مؤقت الحصص

هذا التوثيق خاص بالنسخة التجريبية المستقرة 1.0.

## 1. جدول المدارس: schools

الغرض: حفظ بيانات كل مدرسة وربطها بروابط العرض والإدارة.

أهم الحقول المستخدمة:

| الحقل | الوصف |
|---|---|
| school_slug | الرابط المختصر للمدرسة، وهو المفتاح العملي المستخدم في الروابط |
| school_name | اسم المدرسة الكامل |
| governorate | المحافظة |
| wilayat | الولاية |
| admin_code | رمز مدير المدرسة |
| logo_url | رابط شعار المدرسة |
| app_icon_url | رابط أيقونة التطبيق |
| is_active | حالة المدرسة: مفعلة أو موقوفة |
| primary_color | اللون الرئيسي |
| secondary_color | اللون الثانوي |
| background_color | لون الخلفية |
| theme_style | نمط التصميم |

ملاحظة: يعتمد التطبيق على school_slug وليس على id.

## 2. جدول رسائل المدرسة القديمة/المساندة: school_messages

الغرض: حفظ رسائل المدرسة وبعض الإعدادات النصية الخاصة بالشاشة.

أهم الاستخدامات:

- رسائل الشريط السفلي.
- إعدادات الصندوقين النصيين.
- الإعلانات المجدولة بصيغة تبدأ بـ __SCHEDULED__.
- بعض إعدادات الجداول والتنبيهات في صيغ نصية خاصة.

أهم الحقول:

| الحقل | الوصف |
|---|---|
| school_slug | المدرسة المالكة للرسالة |
| message_text | نص الرسالة أو الإعداد |
| is_active | حالة التفعيل |
| sort_order | ترتيب الرسالة |

## 3. جدول الرسائل الحديثة: school_display_messages

الغرض: إدارة الرسائل التي تظهر في الشاشة وفق منطقة العرض.

أهم الحقول:

| الحقل | الوصف |
|---|---|
| school_slug | المدرسة المالكة للرسالة |
| message_text | نص الرسالة |
| message_type | نوع الرسالة |
| target_area | مكان الظهور: ticker أو all وغيرها |
| sort_order | ترتيب الرسائل |
| is_active | تفعيل أو تعطيل |
| start_date | تاريخ البداية، إن وجد |
| end_date | تاريخ النهاية، إن وجد |
| is_annual | هل تتكرر سنويًا |

## 4. جدول صفوف التوقيت: school_schedule_rows

الغرض: حفظ التواقيت أو صفوف الجدول المرتبطة بالمدرسة.

قد يتضمن:

- اسم الفترة.
- وقت البداية.
- وقت النهاية.
- نوع الفترة.
- الترتيب.
- school_slug.

## 5. جدول إعدادات التنبيهات: school_alert_settings

الغرض: حفظ إعدادات تنبيهات الهاتف أو الحاسوب.

أمثلة:

- تفعيل التنبيهات.
- وقت التنبيه قبل نهاية الحصة.
- نص التنبيه.
- نوع الصوت أو الإشعار.

## 6. جدول سجلات النظام: system_logs

الغرض: حفظ سجل للعمليات المهمة.

أمثلة:

- حفظ الرسائل.
- تعديل إعدادات.
- عمليات حذف.
- عمليات إضافة.

## 7. جداول محتملة أو مساندة

قد توجد الجداول التالية حسب التحديثات:

| الجدول | الاستخدام |
|---|---|
| school_timer_settings | إعدادات إضافية للتطبيق |
| school_devices | أجهزة المدرسة أو شاشات العرض |
| device_activations | تفعيل أجهزة العرض |
| school_theme_settings | إعدادات التصميم والثيم |

## 8. دالة حذف المدرسة

الدالة المستخدمة للحذف النهائي:

```sql
public.delete_school_cascade(
  p_school_slug text,
  p_confirm_slug text,
  p_system_code text
)
```

وظيفتها:

- التحقق من school_slug.
- التحقق من رمز الحذف.
- حذف البيانات المرتبطة.
- حذف المدرسة من جدول schools.
- إرجاع true إذا حُذفت فعليًا.

ملف SQL الخاص بها:

```text
database/delete_school_cascade.sql
```

## 9. ملاحظات أمنية

- يجب تفعيل RLS على الجداول الحساسة.
- عمليات الحذف يجب أن تتم عبر RPC وليس من JavaScript فقط.
- يجب عدم السماح لأي مدرسة بتعديل بيانات مدرسة أخرى.
- يجب فصل صلاحية مدير النظام عن مدير المدرسة.
