# تحسين سرعة فتح التطبيق المثبت على الهاتف

الفرع:

```text
pwa-startup-speed-01
```

## سبب التعديل

بعد اختبار ما بعد الدمج، ظهر أن الموقع سريع على المتصفح في الحاسوب والهاتف، لكن التطبيق المثبت على الهاتف أبطأ عند الفتح.

## التشخيص

بطء التطبيق المثبت يرجع غالبًا إلى طريقة عمل PWA:

1. صفحة البداية كانت تعتمد على الشبكة أولًا عند فتح HTML.
2. Service Worker كان يخدم صفحات HTML بأسلوب network-first.
3. Safari/iPhone كان يتم تنظيف وتعطيل Service Worker بدل تنظيفه ثم تسجيله من جديد.
4. ملفات تشغيل شاشة العرض لم تكن كلها ضمن قائمة الكاش الأولي.

## التعديلات المنفذة

### 1. تحديث sw.js

تم تغيير استراتيجية فتح شاشة المؤقت:

```text
index.html = cache-first ثم تحديث في الخلفية
باقي صفحات HTML = network-first ثم fallback من الكاش
ملفات CSS/JS/images = cache-first ثم تحديث في الخلفية
```

الهدف أن يفتح التطبيق المثبت بسرعة من الكاش، ثم يحدّث الملفات في الخلفية.

### 2. توسيع ملفات الكاش الأساسية

أضيفت ملفات تشغيل شاشة العرض إلى الكاش الأولي، ومنها:

```text
index.html
style.css
ticker-fix.css
mobile-current-row-clean.css
desktop-bg.webp
mobile-bg.webp
supabase-config.js
viewer-multischool-identity.js
script.js
script-optimizations.js
ticker-messages.js
viewer-multischool-message-guard.js
viewer-schedule-sync.js
viewer-alerts-v2.js
pwa-register.js
pwa-school-redirect.js
```

### 3. تحديث pwa-register.js

تم تعديل Safari/iPhone بحيث:

```text
ينظف كاش school-timer القديم مرة واحدة
ثم يسجل Service Worker الجديد
```

بدل تعطيل Service Worker نهائيًا.

### 4. تحديث manifest.json

تم تعديل start_url إلى:

```text
index.html?view=mobile&pwa=1
```

لتمييز فتح التطبيق المثبت عن فتح الرابط العادي.

### 5. تحديث روابط النسخ

تم تحديث إصدارات تحميل ملفات PWA في:

```text
index.html
install.html
```

حتى يلتقط المتصفح النسخة الجديدة بدل الكاش القديم.

## ملاحظات اختبار مهمة

بعد الدمج، قد يحتاج التطبيق المثبت إلى فتحه مرتين:

```text
الفتح الأول: يلتقط Service Worker الجديد وينظف الكاش القديم.
الفتح الثاني: يجب أن يكون أسرع بوضوح.
```

على iPhone قد يكون الأفضل حذف أيقونة التطبيق القديمة وإعادة إضافتها إذا بقي البطء؛ لأن iOS قد يحتفظ بإعدادات start_url القديمة.

## الاختبارات المطلوبة بعد الدمج

1. فتح الرابط العادي في الهاتف.
2. فتح التطبيق المثبت في الهاتف أول مرة.
3. إغلاق التطبيق من التطبيقات المفتوحة.
4. فتح التطبيق مرة ثانية.
5. مقارنة السرعة.
6. التأكد من ظهور اسم المدرسة والشعار والجدول والرسائل.
7. اختبار مدرسة غير مدرسة الشيخ سيف إن وجدت.

## قرار الدمج

لا يُدمج هذا الفرع مباشرة قبل فتح Pull Request ومراجعة الملفات المتغيرة.
