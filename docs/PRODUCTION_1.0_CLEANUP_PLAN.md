# خطة تنظيف ملفات المشروع وتنظيمه كنسخة إنتاجية 1.0

الفرع الخاص بالتنظيف:

```text
cleanup-production-1.0
```

فرع النسخة الاحتياطية قبل التنظيف:

```text
backup-stable-1.0-2026-07-01
```

## القاعدة الأساسية

لا يتم حذف أي ملف مباشرة في المرحلة الأولى. يتم التصنيف أولًا، ثم النقل إلى archive، ثم الاختبار، ثم الحذف لاحقًا عند التأكد.

## أهداف التنظيف

1. فصل ملفات الإنتاج عن ملفات التجارب القديمة.
2. تقليل الملفات المتكررة أو غير المستخدمة.
3. توحيد أسماء الملفات.
4. تسهيل الصيانة المستقبلية.
5. تقليل احتمال تحميل ملفات قديمة أو متعارضة.
6. تجهيز المشروع لمرحلة أمان أعلى وتوسع أكبر.

## خطوات المرحلة الأولى

1. إنشاء هذا الملف.
2. إنشاء ملف جرد تفصيلي.
3. فحص تحميل الملفات من index.html و dashboard-v2.html و system-admin.html و install.html.
4. تحديد الملفات غير المحملة.
5. نقل المرشحين إلى archive بدل حذفهم.
6. اختبار التطبيق بعد النقل.

## اختبارات إلزامية بعد أي نقل

- فتح شاشة الهاتف لمدرسة الشيخ سيف.
- فتح شاشة الهاتف لمدرسة أخرى.
- فتح شاشة الحاسوب.
- فتح لوحة مدير المدرسة.
- حفظ رسالة.
- حفظ إعلان مجدول.
- فتح لوحة مدير النظام.
- إضافة مدرسة تجريبية.
- رفع شعار.
- حذف المدرسة التجريبية.
- تثبيت التطبيق أو فتح صفحة التثبيت.

## ملاحظة

هذا الملف لا يعني أن التنظيف اكتمل. هو بداية مرحلة التنظيم قبل تنفيذ النقل أو الحذف.
